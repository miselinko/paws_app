====================================================
  PAWS APP — PLAN RAZVOJA (2026-03-25)
====================================================

## STANJE PROJEKTA

Funkcionalan MVP sa:
- Django REST backend
- React/TypeScript web app
- React Native/Expo mobilna app (Android)
- Cloudinary za slike
- Groq AI chat bot
- Autentifikacija (JWT)
- Rezervacije, šetači, psi, poruke, recenzije
- Live GPS praćenje šetnje (polling, 5s interval)
- Admin panel (web + mobilni)
- Push notifikacije (Expo)

----------------------------------------------------
## KRITIČNO — pre produkcije
----------------------------------------------------

1. PLAĆANJE
   - Nema Stripe/PayPal integracije
   - Nema naplate, cena, transakcija
   - Procena: 3-5 dana
   - Fajlovi: novi backend app + frontend checkout

2. CORS SIGURNOST
   - CORS_ALLOW_ALL_ORIGINS = True (prima sve domene!) — namerno ostavljeno jer
     backend živi na Renderu i whitelist bi blokirao produkcijski frontend
   - Pravo rešenje kad se zna finalna domena:
       1. U settings.py zameniti sa:
          CORS_ALLOWED_ORIGINS = config('CORS_ALLOWED_ORIGINS',
              default='http://localhost:5173').split(',')
       2. Na Render dashboardu dodati env var:
          CORS_ALLOWED_ORIGINS=https://tvoja-domena.com
   - Fajl: backend/config/settings.py
   - Procena: 30 min (kad je domena poznata)

3. WALKER MOZE SEBI DA POSTAVI is_featured ✅ ZAVRŠENO
   — read_only_fields = ['is_featured'] u WalkerProfileSerializer
   - Admin-only polje nije zaštićeno u WalkerProfileUpdateView
   - Fajl: backend/users/views.py
   - Procena: 30 min

4. NEMA RATE LIMITINGA NA AUTH ✅ ZAVRŠENO
   — login: 10/min, password_reset: 5/h, anon: 200/day, user: 2000/day
   - /auth/login/ — brute force moguć
   - /forgot-password/ — spam emailova moguć
   - Fix: Django REST throttling
   - Procena: 1h

5. EMAIL VERIFIKACIJA ✅ ZAVRŠENO
   — EmailVerificationToken model, VerifyEmailView, ResendVerificationView
   — Frontend /verify-email stranica
   — Migracija 0007 primenjena
   - Ko god može napraviti nalog sa bilo kojim emailom
   - Treba: slanje verifikacionog emaila pri registraciji
   - Procena: 1 dan

## VAŽNO — narednih mesec dana
----------------------------------------------------

6. PUSH NOTIFIKACIJE (mobilni) ✅ ZAVRŠENO
   — PushToken model + PushTokenView
   — send_push_notification() helper u users/views.py
   — Notifikacije se šalju pri: potvrdi/odbijanju rezervacije,
     početku šetnje, završetku šetnje
   - Procena: 1 dan

7. DOSTUPNOST ŠETAČA SE NE PROVERAVA
   - Polje availability postoji ali se ignoriše pri rezervaciji
   - Treba: validacija pri kreiranju rezervacije
   - Fajl: backend/reservations/views.py
   - Procena: 2-3h

8. NEMA OVERLAP PROVERE REZERVACIJA
   - Walker može biti duplo rezervisan u istom terminu
   - Fix: provera preklapanja u serializer/view
   - Procena: 2h

9. PORUKE NEMAJU PAGINACIJU
   - Sve poruke se učitavaju odjednom (1000+ = problem)
   - Fajl: backend/chat/views.py ConversationView
   - Procena: 2h

10. SLIKE — NEMA SIZE LIMIT NI MIME VALIDACIJE ✅ ZAVRŠENO
    — validate_image_file() u users/views.py
    — Max 5MB, dozvoljeni tipovi: JPEG, PNG, WebP, GIF
    - Procena: 1h

## POBOLJŠANJA — kad bude vremena
----------------------------------------------------

11. WALKER DASHBOARD (mobilni) ✅ ZAVRŠENO
    — Prihvatanje/odbijanje rezervacija iz mobilne app
    — Upravljanje rasporedom (availability u ProfileScreen)
    — ReservationDetailScreen sa svim akcijama

12. RECENZIJE NA ŠETAČEVOM PROFILU (mobilni) ✅ ZAVRŠENO
    — ProfileScreen prikazuje sekciju "⭐ Recenzije (N)"
    — Prikazuje prosek, reviewer avatar, ime, datum, zvezdice, komentar

    - Procena: 2-3h

13. DEEP LINKING (mobilni) ✅ ZAVRŠENO
    — scheme: "paws" dodat u app.json
    — LinkingOptions config u RootNavigator.tsx
    — Prefixes: paws:// i https://paws.rs
    — Rute: /rezervacije, /poruke, /profil, /setaci
    - Procena: 1 dan

14. SOFT DELETE KORISNIKA ✅ ZAVRŠENO
    — DeleteAccountView: is_active=False umesto user.delete()
    — AdminUserDetailView.delete: isto, sa AuditLog zapisom
    — Istorija se čuva, nalog se može reaktivirati
    - Procena: 2h

15. DATABASE INDEXI ✅ ZAVRŠENO
    — Reservation.status: db_index=True
    — User.role: db_index=True
    — Message.read: db_index=True
    — Migracije: chat.0002, reservations.0004, users.0009
    - Procena: 1h

16. HAVERSINE NA BAZI (PostGIS) — PRESKOČENO
    - Odluka: Python-side Haversine dovoljno za MVP
    - PostGIS overkill za trenutni obim podataka
    - Revisit kad broj šetača pređe nekoliko stotina
    - Fajl: backend/users/views.py haversine()

17. ADMIN AUDIT LOG ✅ ZAVRŠENO
    — AuditLog model: admin, target_user, action, note, created_at
    — Akcije: ban, unban, feature, unfeature, delete
    — Loguje se automatski u AdminUserDetailView.patch/delete
    — Migracija: users.0009
    - Procena: 1 dan

18. LIVE GPS PRAĆENJE ŠETNJE ✅ ZAVRŠENO
    — IN_PROGRESS status dodat (između confirmed i completed)
    — Polja: last_lat, last_lng, walk_started_at na Reservation
    — WalkStartView: šetač pokreće šetnju (30 min ranog starta)
    — WalkLocationView: GET (čita poziciju) + POST (ažurira poziciju)
    — Mobilni: GPS polling svakih 5s via expo-location
    — Web: Leaflet mapa u ReservationCard, polling svakih 5s
    — Vlasnik vidi poziciju uživo, može otvoriti native Maps
    — Migracija: reservations.0003

19. ADMIN PANEL (mobilni) ✅ ZAVRŠENO
    — AdminScreen sa 5 tabova: Dashboard, Korisnici, Rezervacije, Recenzije, Psi
    — Statistike, pretraga, paginacija, ban/unban, is_featured toggle
    — Vidljiv samo admin korisnicima (🛡️ tab u navigaciji)

20. RECENZIJE — WEB UX FIX ✅ ZAVRŠENO
    — Dugme "⭐ Ostavi recenziju" vidljivo odmah (nije u collapse sekciji)
    — ReviewForm inline u kartici

## TEHNICKE NAPOMENE
----------------------------------------------------

- APK build: powershell.exe -ExecutionPolicy Bypass -File mobile/android/build_release.ps1
- APK izlaz: mobile/android/app/build/outputs/apk/release/app-release.apk
- Mobilni image upload: fetch (ne axios!) + PATCH metoda za profil
- SafeAreaProvider mora biti root wrapper u App.tsx
- Bot zaštita: INJECTION_PATTERNS lista u backend/chat/views.py
- Cloudinary: slike idu u profiles/ i dogs/ foldere
- Nominatim: countrycodes=rs, User-Agent header obavezan
- Deep linking: paws://rezervacije, paws://poruke, paws://profil, paws://setaci
- Soft delete: is_active=False, nalog ostaje u bazi, može se reaktivirati
- AuditLog: backend/users/models.py, loguje ban/unban/feature/delete
